## 一、产品定位

### 1.1 核心价值

让开发者可以通过移动端（微信/飞书/企业微信）远程监控和控制AI编程工具（如Claude Code）的运行过程，实现"随时随地掌握AI编码进度"。

### 1.2 差异化定位

- **不是**手机上写代码
- **不是**通用远程桌面
- **是**AI编程专用移动终端，专注"监控+轻量干预"

## 四、用户体验设计

### 4.1 交互模式

采用**推送+会话**混合模式：
- 关键节点（任务完成、需确认）通过消息推送触达
- 用户在会话中完成交互（按钮确认、快捷回复）

### 4.2 信息架构

三层呈现：
1. **状态层**：任务进度、当前文件、操作类型（一级卡片）
2. **摘要层**：修改文件数、diff概览（二级展开）
3. **详情层**：完整diff、终端输出（下钻H5页面）

### 4.3 视觉编码

| 状态 | 颜色 | 图标 |
|------|------|------|
| 运行中 | 蓝色 | 旋转中 |
| 待确认 | 橙色 | 感叹号 |
| 已完成 | 绿色 | 对勾 |
| 失败 | 红色 | 叉号 |

## 五、技术架构

### 5.1 系统架构

```
┌─────────────────┐     ┌─────────────────┐     ┌─────────────────┐
│   移动端        │     │    云端服务     │     │   桌面端        │
│ (微信/飞书)     │◄───►│   (API网关)     │◄───►│ (Desktop Agent) │
│                 │     │  + WebSocket    │     │                 │
│  - 消息卡片    │     │  + 业务服务      │     │  - 终端IO拦截  │
│  - 菜单按钮    │     │  + 消息队列      │     │  - 指令注入    │
└─────────────────┘     └─────────────────┘     └─────────────────┘
```

### 5.2 核心模块

| 模块 | 技术选型 | 说明 |
|------|---------|------|
| 消息网关 | Node.js/Go | 对接微信/飞书/企业微信API |
| 实时通信 | WebSocket | 终端输出实时推送 |
| 终端拦截 | pty + Node.js | 捕获AI工具stdout/stderr |
| 指令注入 | stdin | 向AI工具发送用户指令 |
| 存储 | Redis + MySQL | 会话状态、用户配置 |

### 5.3 安全设计

- 传输层：TLS 1.3 + mTLS
- 认证：OAuth 2.0 + JWT
- 加密：AES-256-GCM（敏感数据）
- 审计：完整操作日志
